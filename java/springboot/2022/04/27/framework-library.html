프레임워크와 라이브러리

프레임워크

컴퓨터 프로그래밍에서 복잡한 문제를 해결하거나 서술하는데 사용되는 기본 개념 구조이다.

소프트웨어의 구체적인 부분에 해당하는 설계와 구현을 재사용이 가능하게끔 일련의 협업화된 형태로 클래스들을 제공하는 것이다.

장점

  • 효율적이다 -> 아무것도 없는 상태에서 코드를 일일이 짜는 것 보다 시간과 비용이 훨씬 절약되며 생산성이 좋아진다.

  • 퀄리티 향상 -> 버그 발생 가능성을 처리해줌으로써 개발자가 반복 작업에서 실수하기 쉬운 부분을 커버해준다 다수의 개발자가 사용하며 수정한 일종의 검증된 코드라고 볼 수 있다.

  • 유지보수가 쉽다 -> 일일이 코드를 짜 놓은 경우, 개발 담장자가 바뀌면 코드에 손 대는것이 힘들어진다 하지만 프레임 워크를 사용하면 코드가 보다 체계적이기 떄문에 담당자가 바뀌더라도 위험 부담을 줄일 수 있고 유지 보수에 안정적이다.

단점

  • 학습시간이 길다 -> 코드를 본인이 짜 놓은 것이 아니기 때문에, 프레임워크에 있는 코드를 습득하고 이해하는데 오랜 시간이 걸린다.

  • 제작자의 의도된 제약 사항 -> 제작자가 설계한 구조를 어느 정도 유지한 채 코드에 살을 붙여야한다 그렇기 때문에 자유롭고 유연한 개발에 한계가 있다.

스프링 프레임워크

엔터프라이즈 급 애플리케이션 개발에 필요로하는 경량형 프레임워크이다. j2EE에서 제공하는 대부분의 기능을 지원하기 떄문에 자바 개발에 있어서 대표적인 프레임워크로 자리잡고 있으며, JDBC를 비롯하여 iBatis, 하이버네이트, JPA등 DB처리를 위해 널리 사용되는 라이브러리와 연동을 지원하고 있다.

경량컨테이너로 라이프사이클을 관리 필요한 객체를 스프링으로부터 받아온다.

DI지원하여 객체간의 의존관계 설정이 가능하다.

AOP지원한다.

POJO방식으로 자바객체는 특정한 인터페이스를 구현하고 클라스 사속이 필요치 않다.

트랜젝션 처리를 위한 일관된 방법을 제공한다

영속성 관련 다양한 API를 지원한다.

API연동을 지원한다.

라이브러리

라이브러리는 자주 사용되는 로직을 재사용하기 편리하도록 잘 정리한 일련의 코드들의 집합이다.

계산을 위한 라이브러리, D접속을 위한 라이브러리, PDF 텍스트 추출을 위한 라이브러리등 특정 기능을 수행할 떄 필요한 코드들을 모아놓은 것이라 보면 된다.

  • 모든 기능을 직접 코딩 하지 않아도 된다.

라이브러리는 각 기능을 해내는 코드들의 집합이므로 자신이 직접 일일히 작성하지 않아도 필요한 기능이 구현된 라이브러리를 불러와서 사용하면 된다.

  • 시간이 절약된다

위에 말했듯이 이미 기능이 구현된 라이브러리를 이용하면 코딩하는데 들어가는 시간이 크게 줄어들게 된다.

ex) 원, 삼각형, 사각형의 도형 30개를 모두 크기가 다르게 그려야 할 때

라이브러리 없이 코딩할 경우 삼각형 1,2,3…. 사각형 ,1,2,3…. 원 1,2,3…. 같이 일일이 코딩해야 한다.

라이브러리를 이용 하면 삼각형의 경우 세변의 길이 사각형의 경우 네변의 길이 원의 경우 지름의 길이를 입력 하면 각 도형을 입력해주는 라이브러리를 사용하여 손쉽게 그려낼 수있다.

  • 누구나 라이브러리를 작성 할 수 있다.

어떠한 작업을 간편하게 수행하기 위해 코드를 작성했다면 그것이 바로 라이브러리라고 할 수 있다.

프레임워크와 라이브러리의 차이

사용자로 하여금 구현하고자 하는 기능을 쉽게 제공해주는 계층이란 면에서 라이브러리와 프레임워크는 비슷한 점이 있다.

하지만 프레임워크에는 라이브러리에 뼈대가 되는 클래스 들과 클래스들의 관계로 만들어진 설계의 기본 틀이 추가가 된 것이 다르다.

  • 프레임워크 : 설계의 기반이 되는 부분을 기술한 확장 가능한 기반코드와 사용자가 기반코드를 입맛대로 확장하는데 필요한 라이브러리를 통합적으로 제공하는 형태. 기본적인 설계와 라이브러리를 제공해주며 사용자에기 구현하고 싶은 기능 구현에만 전념하게 해준다 다만 제작자가 설계한 구조를 유지해가면서 확장할 수 있도록, 제작자의 의도된 제약 사항이 존재한다.

  • 라이브러리 : 기능을 구현하게 해주는 코드들의 집합. (도구) 기능만을 담아둔 것이기 때문에 동일한 라이브러리 이용해 만든 동일한 기능의 프로그램일지라도 클래스 관계 구조나 데이터 처리절차 품질등이 제각각이다.
